## Description

Chess is one of the largest games worldwide, with over 600 million users. I decided to create this web app as a chess game application that follows the same rules as the original chess game. Each player takes turns playing on an 8 x 8 board, where each player has 16 pieces. The goal of the game is to corner the queen so it can no longer move, also known as checkmate.

## User Stories

- As a user, I want to see a clean user interface.
- As a user, I want to have a good user experience.
- As a user, I want to be able to choose between different time controls for a chess game (5-minute game, 1-minute game, 10-minute game, 3|2-minute game, and so on).
- As a user, I want the app to use the same rules as the original game.
- As a user, I want the board to switch based on whether I am playing as black or white.
- As a user, I want the timer to stop when it is the opponent's turn and start when it is my turn.
- As a user, I want the game to end automatically and decide the winner based on who still has time left on the clock.
- As a user, I want to win the game if my opponent can no longer move their queen, and I should lose if I can no longer move my queen.
- As a user, when I capture an opponent's piece, I want to take its place with the piece I captured with, and I want to earn points depending on the piece: pawn = 1, knight = 3, bishop = 3, rook = 5, queen = 8.
- As a user, I want the pawn to move two squares vertically only on its first move, and then only one square vertically after the first move.
- As a user, I want the pawn to be able to capture an opponent's piece that is one square away diagonally to the right or left.
- As a user, I want the rook to be able to move only horizontally and vertically across the board as long as it has clear squares in its path.
- As a user, I want to capture with the rook if there is an opponent's piece on the rook's path either horizontally or vertically.
- As a user, I want the knight to be able to move in an L-shape (two squares vertically and one square horizontally), and it can jump over other pieces.
- As a user, I want the knight to move to any square; if the square has an opponent's piece, the knight can capture it, but if the square is occupied by one of my own pieces, I cannot move there.
- As a user, I want the bishop to move only diagonally to the right or left, across the board as long as it has clear squares in its path.
- As a user, I want to capture with the bishop if there is an opponent's piece on the bishop's diagonal path.
- As a user, I want the queen to move horizontally, vertically, and diagonally across the board as long as it has empty squares in its path.
- As a user, I want to capture with the queen if there is an opponent's piece on the queen's path horizontally, vertically, or diagonally.
- As a user, I want the king to be able to move only one square in any direction, as long as the square is not attacked by an opponent's piece.
- As a user, I want to capture with the king if there is an opponent's piece adjacent to it, as long as capturing that piece does not place my king in check.
